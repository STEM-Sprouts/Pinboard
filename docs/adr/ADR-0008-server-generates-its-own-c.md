# ADR-0008 — The compile server generates its own C

**Status:** Accepted

## Context

A live compile endpoint that accepts client code strings is an arbitrary-C++-execution service. A malicious client can send anything and claim it was generated by Pinboard; string scanning is not a security boundary.

## Decision

The compile server accepts only a **project document**, lowers it to IR and prints the Arduino C **itself**, then compiles only that (`docs/domains/compiler.md`). It never compiles a client-submitted string.

## Consequences

- The lowering + printer code must be runnable server-side.
- Worker constraints apply: allowlisted boards/libraries, no network, isolated temp dir, timeouts, size limits, rate-limiting by user/session (not IP alone — classrooms share NAT).
- The endpoint's exit test: it *cannot* compile an arbitrary client string.
